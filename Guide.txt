1, Subnetting, gán IP cho router
B1: Chọn Router, bật CLI mode
B2: gõ các lệnh để vào chế độ config
enable
show ip route (để xem các tuyến địa chỉ nào chưa được nối, rồi định tuyến cho nó)
config terminal
interface <interface (f0/0, s0/0,....)
ip address <ip muốn gán> <Subnet Mask của ip đó>
(Nếu chọn interface là s0/0 - có hình đồng hồ thì thêm câu lệnh sau: "clock rate 56000")

2, Cấu hình VLAN
B1: Chọn Switch, vào CLI mode
B2: Gõ lệnh sau để thêm cổng vlan:
enable
configure terminal
hostname <tên tùy ý>
vlan <số đề bài yêu cầu>
<Muốn cài thêm vlan khác thì "exit" ra rồi cài tiếp vlan>
exit
B3: Set interface cho từng VLAN 
interface <f0/1, f0/2,...> 
<Với các interface nối với PC thì tiếp theo dùng lệnh "switchport access vlan <số VLAN muốn gán>">
<Với interface nối với Router thì tiếp theo dùng lệnh "switchport mode trunk">
B4: Set IP cho Router kết nối vs Switch, chọn vào router -> CLI
enable
configure terminal
hostname <Tên>
interface <interface (f0/0, s0/0,....)
<Nếu nối với Switch, thì chỉ thêm "no shutdown" vào interface đó rồi "exit", tiếp theo
	interface <interface./<số VLAN> (VD: f0/0.10 -> VLAN 10)
	encapsulation dot1Q <số VLAN>
	ip address <ip> <SM>
<Nếu ko nối với Switch thì cứ interface đến các phần tử khác rồi set ip address -> no shut>

5, Cấu hình DCHP cho Router
B1: Vào configure terminal
B2: ip dhcp <tên Poo>
B3: network <NetID> <SubnetMask>
B4: default-router <IP router>
B5: dns-server <Địa chỉ dns>
B6: domain-name <Tên domain>
Nếu muốn DHCP loại trừ 10 địa chỉ đầu tiên => exit rồi ip dhcp excluded-address <d/c đầu> <d/c cuối>
Nếu muốn DHCP trên từng VLAN thì làm y như này, chỉ thay các địa chỉ ở B4 và B5 sao cho tương ứng với từng VLAN

6, Nhiều Switch Kết nối với nhau
B1: Config VLAN 1 switch với Router như bình thường, bật mode trunk các cổng kết nối với switch này
B2: Các switch khác nối với Switch này chỉ cần bật mode trunk ở các cổng nối với switch này, rồi bật mode access VLAN tương ứng cho các VLAN đã cài ở B1 cho từng cổng trên các Switch đó

8, Cấu hình NAT
Giả sử có 1 đám OSPF đằng trước và 1 con Router 4 kết nối với Router NAT đằng sau để biên dịch địa chỉ ra bên ngoài
B1: Cấu hình tất cả OSPF, kể cả cho Router NAT
B2: Vào chế độ router ospf của Router NAT, rồi nhập câu lệnh default-information originate (ospf), redistribute-static(eigrp) => chuyển đổi lường ospf từ mạng cục bộ sang 1 mạng bên ngoài
B3: Định tuyến tĩnh cho Router NAT ra bên ngoài với IP mặc định bất kỳ định tuyến tới là 0.0.0.0 SM 0.0.0.0 <next-hop> sẽ là địa chỉ outside của con R4 kết nối với Router NAT
B4: Cấu hình con ROuter4, với cổng ra NAT là ip nat outside, cổng vào miền ospf cục bộ là ip nat inside
B5: Cấu hình NAT trên R4
-> Nễu là Static NAT (chuyển từng private address sang public address) => ip nat inside source static <priv> <publ>
-> Nếu là Dynamic NAT (tự động chuyển linh hoạt) 
	=> Cấu hình ACL (access-list, ip access-list ...), 
	=> Tạo pool NAT (ip nat pool <Tên> <d/c bắt đầu> <d/c kết thúc> netmask <SM>)
	=> ip nat inside source list <ACL> pool <Tên>
-> Nếu là NAT OVERLOAD (PAT) (đ/c private nhiều hơn đ/c public)
	=> Cấu hình ACL (access-list, ip access-list ...), 
	=> Tạo pool NAT (ip nat pool <Tên> <d/c bắt đầu> <d/c kết thúc> netmask <SM>)
	=> ip nat inside source list <ACL> pool <Tên> interface <cổng ra NAT outside> overload
